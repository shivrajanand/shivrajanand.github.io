NOTES

  • Slide 2
    • Dropbox optimized speed and concurrency by rewriting performance-critical components in Java and Go while keeping Python for many services.
    • Twitter migrated from Ruby on Rails and Python to Java to improve scalability and handle millions of concurrent users efficiently.
    • Apache Beam supports both Java and Python for building batch and streaming data pipelines across engines like Apache Flink and Google Cloud Dataflow.
    • Py4J enables Python applications to interact dynamically with Java, used by companies like Netflix, Uber, and Airbnb.
    • Jython allows Python to run on the Java Virtual Machine (JVM), helping enterprises integrate Python into Java-based systems without major rewrites.
  • Slide 5
    • AVATAR fine-tunes the Transcoder, which is trained using unsupervised algorithms on GitHub monolingual codebases, leveraging parallel codes written in Java and Python.
    • Parallel Code Pairs refer to programming solutions available in both Java & Python. A single pair is called a Java-Python Parallel Code Pair.
    • Unit Test: A small, automated test that checks whether a specific function, method, or module in a program works correctly. It tests a single unit of code in isolation to ensure expected behavior.
  • Slide 9
    • N1: The open source library difflib was used to compare all the solutions and select five that differs most from others. [difflib is a builtin Python module used for comparing sequences (like strings, lists, or files) and identifying differences between them. It is commonly used for text comparison, version control, and code similarity checks.]
    • N2: Since 15 solution are taken for each problem in both languages we have a total of 125 parallel examples for training. AVATAR relies on multiple ground truths that means for a particular problem there can be multiple possible solution with different logic and styles. Many evaluation metrics were used to evaluate these solutions.
  • Slide 10
    • Parallel Functions provide a direct one-to-one mapping between Java and Python implementations.
    • These function pairs help train machine learning models for code translation.
    • Used for evaluation to ensure that models generate functionally correct translations.
    • AVATAR’s parallel functions are tested against the dataset from Lachaux et al. (2020), which introduced TransCoder, an unsupervised model for Java-Python-C++ translation.
    • AVATAR compares its translated functions with those generated by TransCoder to measure improvements in translation accuracy.
  • Slide 11
    • BLEU Score (Bilingual Evaluation Understudy Score)
      • Measures textual similarity between the translated code and the reference solution.
      • Calculates the overlap of tokens (words, symbols, or code elements) between the two versions.
      • Higher BLEU scores indicate a closer match to the reference translation.
      • Example: int a + b = c and num1 + num2 = total will have a high BLEU score because most tokens match.
    • SM Score (Syntax Matching Score)
      • Checks how much of the translated code's Abstract Syntax Tree (AST) matches the AST of the reference program.
      • AST represents the structural hierarchy of a program (e.g., loops, conditionals, method calls).
      • A high SM score means the syntactic structure is well preserved.
      • Example: If the reference code uses a for loop and the translated code also uses a for loop instead of a while loop, it will have a higher SM score.
    • DM Score (Dataflow Match Score)
      • Evaluates whether the dataflow in the translated code is preserved correctly.
      • Dataflow refers to how variables are assigned, modified, and used throughout the program.
      • The score is calculated as the ratio of matched data flows in the candidate code to those in the reference code.
      • Example: If a function in Java initializes sum = 0 and updates it inside a loop, but the translated Python function initializes sum = 0 but updates it incorrectly, the DM score will be lower.
    • CB Score (CodeBLEU Score)
      • An advanced evaluation metric for programming languages.
      • Combines BLEU, Syntax Match (SM), and Dataflow Match (DM) into a weighted score.
      • More effective than BLEU alone because it considers both syntax and logical correctness.
      • Example: A translation with a high BLEU score but incorrect data flow will still score poorly on CodeBLEU.
    • EA Score (Execution Accuracy Score)
      • Measures whether the translated program can execute successfully without errors.
      • Checks for syntax errors, missing imports, and runtime crashes.
      • A high EA score means the translation is syntactically valid and compiles without issues.
      • Example: If Java code using BufferedReader is translated to Python but lacks the import sys statement, the EA score drops because it fails to execute.
    • CA Score (Computational Accuracy Score)
      • Proposed by Lachaux et al. (2020), CA measures whether the translated program produces the correct outputs when given the same inputs.
      • Unlike EA, which just checks execution, CA ensures the functional correctness of the translation.
      • A program may execute successfully but still give incorrect results, leading to a low CA score.
      • Example: If Java code correctly calculates Fibonacci numbers, but the translated Python version implements the logic incorrectly, the CA score will be low.
